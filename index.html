1. OpenServer орнотуу жана биринчи скрипт

1.1. Практиканын максаты

Бул практикалык сабакта студенттер төмөнкү көндүмдөргө ээ болушат:

  • OpenServer программасын өз компьютерине туура орнотуу.
  • Жергиликтүү сервердин ишин түшүнүү (Apache, PHP, MySQL).
  • Биринчи PHP скриптти (Hello World) түзүп иштетүү.
  • Серверди баштапкы деңгээлде тестирлөө.

1.2. OpenServer деген эмне?

OpenServer бул Windows операциялык системасы үчүн даяр локалдык сервердик платформа. Ал Apache/Nginx веб-серверин, PHP интерпретаторун, MySQL/MariaDB маалымат базасын, phpMyAdmin башкаруу куралын жана башка керектүү компоненттерди камтыйт.

Артыкчылыктары:

  • Windowsко оңой орнотулат.
  • PHP версияларын каалаганча өзгөртүп колдонууга болот.
  • MySQL/MariaDB базасын башкаруу ыңгайлуу.
  • phpMyAdmin камтылган.
  • Бир нече долбоор менен иштөөгө ылайыктуу.

1.3. OpenServer орнотуу кадамдары

  1. 1-кадам: Программаны жүктөө - https://ospanel.io/ расмий сайтына кирип, программанын актуалдуу версиясын жүктөп алыңыз.
  2. 2-кадам: Орнотуу файлын иштетүү - Жүктөлгөн архивди же орнотуучу файлды ачып, аны тутумдук дисктин түпкү папкасына (мисалы, C:\OpenServer\) көчүрүңүз.
  3. 3-кадам: Программаны иштетүү - Папканын ичиндеги OpenServer.exe файлын администратордун укугу менен иштетиңиз. Экрандын төмөнкү оң бурчунда (трейде) кызыл желекче пайда болот.
  4. 4-кадам: Жөнгө салуу - Настройки (Жөндөөлөр) бөлүмүнө кирип, модулдар өтмөгүнөн Apache веб-серверин, PHP версиясын жана MySQL маалымат базасын тандап алыңыз.
  5. 5-кадам: Серверди иштетүү - Желекчени басып, "Запустить" буйругун тандаңыз. Желекче кызылдан сарыга, андан соң жашыл түскө өзгөрүшү керек. Жашыл түс — сервер ийгиликтүү иштеп жатканын билдирет.

1.4. Биринчи PHP долбоорун түзүү

  1. 1-кадам: Сервердин ички папкасындагы C:\OpenServer\domains\ дарегине өтүп, жаңы test.loc деген папка түзүңүз.
  2. 2-кадам: Түзүлгөн папканын ичине тексттик редактор (мисалы, Notepad++, VS Code) аркылуу index.php деген файл ачып, төмөнкү кодду жазыңыз:
<?php
echo "Hello, World!";
?>
  1. 3-кадам: Браузериңизди ачып, дарек тилкесине http://test.loc/ деп жазып, Enter баскычын басыңыз. Экранда "Hello, World!" деген жазуу пайда болот.

1.5. Hello World деген эмне?

"Hello World" — программалоо дүйнөсүндө кабыл алынган, каалаган программалоо тилин үйрөнүүдөгү эң алгачкы кадам. Анын негизги максаты — татаал код жазуу эмес, тандалган чөйрөнүн, интерпретатордун жана сервердин туура конфигурацияланып, катасыз иштеп жаткандыгын текшерүү.

1.6. Жергиликтүү серверди текшерүү

Сервер жана анын ички компоненттери тууралуу толук маалымат алуу үчүн PHP тилинин атайын камтылган phpinfo() функциясын колдонобуз. Ал үчүн файлга төмөнкү кодду жазып иштетебиз:

<?php
phpinfo();
?>

1.7. Жаңылануучу каталар жана алардын чечилиши

  1. Сервер иштебейт (желекче кызыл бойдон калат): Көбүнчө компьютердеги башка программалар (мисалы, Skype, VMware) 80 же 443 портторун ээлеп алганда болот. Чечүү жолу: Башка программаларды жабуу же OpenServer жөндөөлөрүнөн портту өзгөртүү.
  2. Баракча ачылбайт (404 же 500 катасы): Сиз түзгөн файл туура эмес папкада сакталган же кеңейтүүсү .php эмес, .txt болуп калган. Файлдын жолун жана атын кайра текшериңиз.

2. PHP конфигурациясы жана биринчи коддор

2.1. PHP орнотуу жана иштетүү

PHP (Hypertext Preprocessor) — веб-серверде аткарылуучу, динамикалык веб-баракчаларды жана татаал веб-тутумдарды түзүү үчүн иштелип чыккан кеңири таралган скрипттик программалоо тили. Ал сервер тарапта (Backend) иштеп, натыйжасын колдонуучуга таза HTML түрүндө кайтарат.

Жогоруда айтылгандай, Windows чөйрөсүндө PHP менен иштөөнүн эң оңой жолу — бул OpenServer платформасы. Анын башкы менюсунан долбоордун талабына ылайык PHP'дин каалаган версиясын (мисалы, PHP 7.4, 8.0 же 8.2) оңой которуп алсак болот.

Практикалык көнүгүү (Конфигурациялоо):
  1. Сервердин жөндөөлөрүнөн php.ini (негизги конфигурациялык файл) файлын ачыңыз.
  2. Иштеп чыгуу процессинде каталарды экранга чыгаруу үчүн display_errors = On жана error_reporting = E_ALL саптарын таап, күйгүзүңүз.
  3. Өзгөртүүлөр күчүнө кириши үчүн OpenServerди кайра жүктөңүз (Перезапустить).

2.2. PHP скрипт түзүү жана иштетүү

Мисал 2.1: Өзгөрмөлөр менен иштөө

PHPде өзгөрмөлөр дайыма доллар ($) белгиси менен башталат. Төмөнкү кодду жазып текшериңиз:

<?php
$name = "Uluk";
$age = 25;
echo "Менин атым " . $name . " жана мен " . $age . " жаштамын.";
?>

Натыйжа: Менин атым Uluk жана мен 25 жаштамын.

2.3. PHP менен HTML бириктирүү

PHP кодун түз эле HTML тегдеринин ичине интеграциялоого болот. Бул веб-баракчанын белгилүү бир бөлүктөрүн динамикалык кылууга мүмкүнчүлүк берет:

<!DOCTYPE html>
<html>
<head><title>PHP жана HTML</title></head>
<body>
    <h1><?php echo "Салам, бул PHP аркылуу басылды!"; ?></h1>
    <p><?php echo "Бул PHP жана HTML коддорун бириктирүүнүн жөнөкөй мисалы."; ?></p>
</body>
</html>

2.4. Формаларды иштетүү негиздери

Колдонуучу киргизген маалыматтарды серверде кабыл алуу веб-программалоонун өзөгү болуп саналат. Төмөндө эки файлдын өз ара байланышы көрсөтүлгөн:

form.html (Фронтенд бөлүгү):

<form method="POST" action="process.php">
    Атыңызды киргизиңиз: <input type="text" name="name">
    <input type="submit" value="Жөнөтүү">
</form>

process.php (Бэкенд аткаруучу бөлүгү):

<?php
$name = $_POST['name'];
echo "Сиздин атыңыз: " . htmlspecialchars($name);
?>

2.5. Веб-сервер жана логдор менен иштөө

Скрипттердин аткарылуусунда ката кетсе, Apache же Nginx веб-сервери атайын error.log файлына жазып турат. Бул логдорду карап, программанын кайсы сабында кандай ката кеткендигин оңой аныктоого болот.

2.6. PHP менен файлдарга маалымат жазуу

<?php
$file = "data.txt";
file_put_contents($file, "Салам, дүйнө!\n", FILE_APPEND);
echo "Файлга маалымат жазылды. Учурдагы мазмуну: <br>";
echo nl2br(file_get_contents($file));
?>

3. Өзгөрмөлөр жана маалымат түрлөрү

Бул бөлүмдө биз PHP тилинин негизги синтаксисин, өзгөрмөлөрдүн эрежелерин, стандарттык маалымат түрлөрүн, константаларды түзүүнү жана комментарийлерди колдонууну үйрөнөбүз.

3.1. PHP кодунун негизги түзүлүшү

PHP коду ар дайым <?php жана ?> тегдеринин ортосунда жазылат. Бул тегдер серверге коддун кайсы жеринен баштап PHP катары интерпретациялоо керектигин билдирет. Ар бир өзүнчө оператордун (буйруктун) аягына сөзсүз түрдө чекиттүү үтүр (;) коюлат.

3.2. Өзгөрмөлөр (Variables) жана алардын эрежелери

PHPде өзгөрмөлөр маалыматтарды убактылуу сактоочу идиш сыяктуу. Аларды түзүүдө төмөнкү эрежелерди сактоо керек:

  • Өзгөрмө дайыма $ белгиси менен башталат (мисалы: $x).
  • Өзгөрмөнүн аты тамга же ылдыйкы сызык (_) менен гана башталышы керек.
  • Өзгөрмөнүн аты сан менен башталбашы керек (мисалы: $1var — ката).
  • Атта англис тамгалары, сандар жана ылдыйкы сызык гана колдонулат. Регистрге сезимтал келет ($Age жана $age — эки башка өзгөрмө).

3.3. Негизги маалымат түрлөрү (Data Types)

PHP динамикалык түрдө типтерди аныктоочу тил (loosely typed language). Өзгөрмөгө кандай маани берсеңиз, анын тиби автоматтык түрдө ошого өзгөрөт:

  1. String (Текст): Тырмакчанын ичине алынган каалаган текст. $title = "PHP Сабагы";
  2. Integer (Бүтүн сан): Бөлчөк эмес кадимки сандар. $count = 150;
  3. Float / Double (Ондук бөлчөк сан): Үтүр/чекит менен жазылган сандар. $price = 45.50;
  4. Boolean (Логикалык тип): Болгону эки гана маанини кабыл алат: true (чын) же false (калп).
  5. Array (Массив): Бир өзгөрмөдө бир нече маанини топтоп сактоо куралы. $colors = ["Кызыл", "Жашыл", "Көк"];
  6. NULL (Мааниси жок): Өзгөрмө бар, бирок анын ичинде эч кандай маалымат жок экендигин билдирет.

Маалымат тибин аныктоо (var_dump):

Өзгөрмөнүн маанисин жана анын тибин толук көрүү үчүн var_dump() функциясы колдонулат:

<?php
$var = NULL;
var_dump($var); // Натыйжа: select NULL

$var = "Salam";
unset($var); // unset() өзгөрмөнү эс тутумдан толугу менен жок кылат
var_dump($var); // Notice: Undefined variable катасын берет
?>

3.4. Константалар (Constants)

Константа — бул программа аткарылып жаткан учурда мааниси эч качан өзгөрбөй турган туруктуу чоңдук. Аны түзүүнүн эки жолу бар:

  • define('SITE_NAME', 'Менин Веб-сайтым');
  • const PI = 3.14159;

Алдын ала аныкталган константалар: PHPдин ичинде даяр туруктуулар бар. Мисалы, __FILE__ учурдагы файлдын толук жолун көрсөтсө, __LINE__ ошол код жазылган саптын номерин берет.

3.5. Комментарийлердин түрлөрү

Комментарийлер — бул коддун ичиндеги программист үчүн жазылган эскертүүлөр, аларды сервер аткарбайт жана браузерге көрүнбөйт:

  • // Бул бир саптык комментарий
  • # Бул дагы бир саптык комментарийдин түрү
  • /* Бул бир нече саптан турган узун комментарий блогу */

4. Операторлор жана шарттуу конструкциялар

Ар бир программалык камсыздоонун негизги максаты — маалыматтарды кабыл алып, аларды арифметикалык же логикалык операторлор аркылуу иштетип, белгилүү бир шарттарга жараша ар кандай чечимдерди кабыл алуу болуп саналат.

4.1. Арифметикалык операторлор

Математикалык эсептөөлөрдү жүргүзүү үчүн төмөнкү стандарттык операторлор колдонулат:

Оператор Аткарган аракети Мисал Натыйжа
+ Эки маанини кошуу 5 + 3 8
- Биринчи мааниден экинчисин алуу 10 - 4 6
* Көбөйтүү амалы 6 * 2 12
/ Бөлүү амалы 15 / 3 5
% Бөлүүдөн калган калдык (modulo) 10 % 3 1
++ Инкремент (маанини 1ге көбөйтүү) $x++ -

4.2. Салыштыруучу операторлор

Эки маанини өз ара салыштырып, жыйынтыгында логикалык (true/false) маани кайтарат:

  • == : Маанилери барабар болсо (түрүнө карабайт). Мисалы, "5" == 5 чындык (true) чыгарат.
  • === : Идентично (мааниси да, маалымат тиби да барабар болушу керек). "5" === 5 жалган (false) чыгарат.
  • != же <> : Барабар эмес.
  • >, <, >=, <= : Чоң, кичине, чоң же барабар, кичине же барабар.
<?php
$x = 10; $y = 20;
var_dump($x == $y); // bool(false)
var_dump($x != $y); // bool(true)
var_dump($x < $y);  // bool(true)
?>

4.3. Логикалык операторлор

Бир нече шарттарды бириктирип, татаал логикалык чынжырларды түзүүдө колдонулат:

  • && (же and) - Логикалык "ЖАНА". Бардык шарттар туура (true) болгондо гана жалпы жыйынтык true болот.
  • || (же or) - Логикалык "ЖЕ". Бириктирилген шарттардын жок дегенде бирөөсү туура болсо, жыйынтык true болот.
  • ! - Логикалык "ЭМЕС" (Инверсия). Маанини карама-каршысына өзгөртөт.
<?php
$a = 5; $b = 10;
var_dump($a > 0 && $b > 0); // Натыйжа: bool(true)
var_dump(!($a > $b));       // Натыйжа: bool(true), анткени 5 чоң эмес 10дон
?>

4.4. Шарттуу конструкциялар (if / else / elseif)

Программанын логикалык багытын өзгөртүүчү эң негизги конструкция:

<?php
$age = 18;
if ($age >= 21) {
    echo "Сиз толук укуктуусуз.";
} elseif ($age >= 18) {
    echo "Сиз бойго жеткенсиз, бирок чектөөлөр бар.";
} else {
    echo "Сиз жете элексиз.";
}
?>

5. Циклдер жана массивдер

Кайталанма процесстерди автоматташтыруу үчүн циклдер, ал эми чоң көлөмдөгү окшош маалыматтарды бир жерде структуралаштырып сактоо үчүн массивдер колдонулат.

5.1. Циклдердин түрлөрү (Loops)

1. While циклы: Шарт качанга чейин туура (true) болуп турса, циклдин ичиндеги код кайталана берет.

<?php
$x = 1;
while ($x <= 5) {
    echo "Сан: $x <br>";
    $x++;
}
?>

2. Do...While циклы: Бул циклде код эң аз дегенде бир жолу сөзсүз аткарылат, андан кийин гана шарт текшериле баштайт. Бул көбүнчө колдонуучудан пароль суроодо ыңгайлуу.

<?php
$count = 0;
do {
    echo "Бул аткарылган кадам: $count <br>";
    $count++;
} while ($count < 3);
?>

3. For циклы: Кайталануулардын саны алдын ала так белгилүү болгон учурда колдонулуучу эң популярдуу цикл.

5.2. Массивдер (Arrays)

PHPде массивдердин үч түрү бар:

  1. Индекстелген массив: Элементтери кадимки 0, 1, 2... сандары менен индекстелет.
  2. Ассоциативдүү массив: Индекстердин ордуна атайын аталыштар (ачкычтар - keys) колдонулат.
  3. Көп өлчөмдүү массив: Массивдин ичинде дагы башка массивдер жайгашкан татаал структура.

Мисал: Ассоциативдүү массивди Foreach аркылуу айланып чыгуу

<?php
$grades = ["Математика" => 5, "Физика" => 4, "Информатика" => 5];
$sum = 0;
foreach ($grades as $subject => $grade) {
    echo "$subject сабагынан алган баасы: $grade <br>";
    $sum += $grade;
}
$average = $sum / count($grades);
echo "Орточо баасы: " . $average;
?>

$n = 10; $a = 0; $b = 1;

6. PHP Функциялар

Функция — бул программанын ичиндеги өзүнчө обочолонгон, белгилүү бир конкреттүү тапшырманы аткарууга багытталган жана каалаган учурда кайталап чакырып колдонууга мүмкүн болгон коддордун блогу. Ал коддун кайталануусун азайтат.

6.1. Функциялардын классификациясы:

  1. Колдонуучу тарабынан түзүлгөн функциялар: Программист өз муктаждыгына жараша function сөзү менен өзү түзөт.
  2. Ички стандарттык функциялар: PHPдин өзүндө даяр орнотулган функциялар (мисалы, strlen(), count(), date()).
  3. Анонимдүү функциялар (Closure): Атайын аты жок, өзгөрмөлөргө байланып же башка функциянын ичинде аргумент катары түзүлүүчү функциялар.
  4. Рекурсивдик функциялар: Өзүнүн ичинен өзүн кайра чакырган функциялар (мисалы, факториалды эсептөөдө).

6.2. Жөнөкөй функция түзүү жана чакыруу мисалы

<?php
function salomBer() {
    echo "Салам, окуу куралына кош келиңиз!";
}
salomBer(); // Функцияны чакыруу процесси
?>

7. Формалар жана колдонуучу менен өз ара аракеттенүү

Бул бөлүмдө веб-сайттын коноктору менен түз байланыш түзүүнү, HTML формалары аркылуу маалыматтарды серверге коопсуз өткөрүп берүүнү жана аларды текшерүүнү үйрөнөбүз.

7.1. Тексттик маалыматтарды кабыл алуучу жөнөкөй форма

Колдонуучу өз атын жана почтасын киргизе турган HTML форманын түзүлүшү (form.html):

<form action="process.php" method="post">
    <label for="username">Колдонуучу аты: </label>
    <input type="text" id="username" name="username"><br><br>
    <label for="email">Электрондук почта: </label>
    <input type="email" id="email" name="email"><br><br>
    <input type="submit" value="Жөнөтүү">
</form>

7.2. Татаал форма элементтери (Radio, Checkbox) менен иштөө

Бир нече тандамалдары бар элементтерди PHPде кабыл алууда массивдер колдонулат. Мисалы:

<form method="POST" action="multi_process.php">
    Атыңыз: <input type="text" name="name"><br>
    Жынысы: 
    <input type="radio" name="gender" value="male"> Эркек
    <input type="radio" name="gender" value="female"> Аял<br><br>
    Кызыгуулары:
    <input type="checkbox" name="hobbies[]" value="спорт"> Спорт
    <input type="checkbox" name="hobbies[]" value="музыка"> Музыка
    <input type="checkbox" name="hobbies[]" value="саякат"> Саякат<br><br>
    <input type="submit" value="Маалыматты иштетүү">
</form>

Маалыматты кабыл алуучу Бэкенд код (multi_process.php):

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = trim($_POST['name'] ?? '');
    $gender = $_POST['gender'] ?? 'Белгиленген жок';
    $hobbies = $_POST['hobbies'] ?? [];

    echo "
Кабыл алынган маалыматтар:
";
    echo "Аты: " . htmlspecialchars($name) . "<br>";
    echo "Жынысы: " . $gender . "<br>";
    echo "Кызыгуулары: " . implode(", ", $hobbies);
}
?>

Эскертүү: ?? оператору эгер маани келбей калса ката бербей, бош маани же дефолт маани ыйгарууга жардам берет. implode() функциясы массивди текстке айлантып берет.

8. PHP Сессия жана Cookies

Бул практикалык сабакта биз браузерлер менен сервердин ортосундагы байланышты үзбөй сактоо, колдонуучуларды таануу (авторизация) куралдарын карап чыгабыз.

8.1. Cookies деген эмне жана аны колдонуу?

Cookies — бул сервер тарабынан колдонуучунун компьютерине (браузердин ичине) кичинекей текст түрүндө сакталып коюлуучу маалыматтар. Ал кийинки жолу сайтка киргенде автоматтык түрдө кайра серверге жөнөтүлүп турат.

<?php
// Cookie түзүү: "user" атына "Uluk" маанисин 1 саатка (3600 секунд) сактоо
setcookie("user", "Uluk", time() + 3600, "/");

// Cookie маанисин окуу
if(isset($_COOKIE['user'])){
    echo "Табылган Cookie колдонуучусу: " . $_COOKIE['user'];
} else {
    echo "Мындай Cookie табылган жок!";
}
?>

8.2. Cookie файлын жок кылуу (Өчүрүү)

Cookie файлын өчүрүү үчүн анын сакталуу мөөнөтүн өтүп кеткен убакытка (минуска) тууралап коюу жетиштүү:

<?php
setcookie("user", "", time() - 3600, "/");
echo "Cookie системадан ийгиликтүү өчүрүлдү!";
?>

8.3. Сессия (Session) жана авторизация куруу

Сессия маалыматтарды браузерде эмес, сервердин өзүндө коопсуз сактайт. Ал үчүн ар бир барактын башында session_start() функциясы чакырылышы шарт.

Авторизация кодунун мисалы (Логин системасы):

<?php
session_start();

// Колдонуучу логин жана паролду туура киргизди деп эсептейли
if (isset($_POST['login']) && $_POST['password'] == '1234') {
    $_SESSION['user'] = $_POST['login']; // Сессияга жазуу
    echo "Кош келиңиз, " . $_SESSION['user'] . "!";
} else {
    echo "Кирүү катасы: Логин же пароль туура эмес!";
}
?>

8.4. Колдонуучунун кирүү статусун текшерүү

Сайттын жабык барактарын коргоо үчүн төмөнкүдөй текшерүү жүргүзүлөт:

<?php
session_start();
if(isset($_SESSION['user'])){
    echo "Авторизациядан өткөн: " . $_SESSION['user'];
} else {
    die("Кирүүгө тыюу салынат! Алгач сайтка катталыңыз.");
}
?>

9. PHP Файлдар менен иштөө

Сервердеги файлдык система менен иштөө: файлдарды түзүү, тексттерди жазуу, сапма-сап окуу жана колдонуучудан сүрөт же документтерди кабыл алуу.

9.1. Файл түзүү жана ага маалымат кошуу

Файлдар менен иштөөдө атайын режимдер колдонулат: "w" (жазуу үчүн ачуу, эски мазмунду өчүрөт), "r" (окуу үчүн), "a" (маалыматты аягына кошуу).

<?php
$file = fopen("example.txt", "w");
if ($file) {
    fwrite($file, "Бул файлга жазылган биринчи сап.\n");
    fwrite($file, "Бул экинчи динамикалык сап.\n");
    fclose($file); // Ачылган файлды жабуу милдеттүү
    echo "Файл ийгиликтүү түзүлдү!";
} else {
    echo "Ката: Файлды ачууга мүмкүн эмес!";
}
?>

9.2. Файлдан маалыматтарды сап боюнча окуу

<?php
$file = fopen("example.txt", "r");
if ($file) {
    echo "
Файлдын мазмуну:
";
    while(!feof($file)) { // feof - файлдын акырына жеткенче иштейт
        $line = fgets($file); // fgets - бир сапты окуйт
        echo $line . "<br>";
    }
    fclose($file);
}
?>

9.3. Серверге файл жүктөө (File Upload)

Файл жүктөй турган формада сөзсүз түрдө enctype="multipart/form-data" атрибуту болушу керек:

<form action="upload.php" method="post" enctype="multipart/form-data">
    Сүрөт тандаңыз (JPG/PNG): 
    <input type="file" name="fileToUpload"><br><br>
    <input type="submit" value="Серверге жүктөө">
</form>

Файлды кабыл алуучу Бэкенд скрипт (upload.php):

<?php
$target_dir = "uploads/";
$target_file = $target_dir . basename($_FILES["fileToUpload"]["name"]);
$imageFileType = strtolower(pathinfo($target_file, PATHINFO_EXTENSION));

// Баштапкы коопсуздук текшерүүсү
if($imageFileType != "jpg" && $imageFileType != "png") {
    echo "Ката: Тек гана JPG жана PNG форматтарына уруксат берилген!";
} else {
    // Убактылуу сакталган жерден негизги папкага жылдыруу
    if (move_uploaded_file($_FILES["fileToUpload"]["tmp_name"], $target_file)) {
        echo "Файл ийгиликтүү жүктөлдү: " . htmlspecialchars(basename($_FILES["fileToUpload"]["name"]));
    } else {
        echo "Кечириңиз, файлды жүктөөдө ички ката кетти.";
    }
}
?>

10. PHP жана MySQL менен иштөө

Электрондук окуу куралынын акыркы бөлүмүндө биз эң маанилүү теманы — PHP программалоо тилин MySQL маалымат базасы менен байланыштырууну карап чыгабыз.

10.1. MySQL маалымат базасында таблица түзүү (SQL суроо)

Алгач phpMyAdmin куралына кирип же SQL буйругу аркылуу төмөнкү базаны жана таблицаны түзүп алабыз:

CREATE DATABASE test_db;
USE test_db;

CREATE TABLE users (
    id INT(6) UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    age INT(3) NOT NULL
);

10.2. PHP аркылуу Маалымат базасына туташуу (MySQLi объекттик ыкма)

<?php
$servername = "localhost";
$username = "root";
$password = ""; // Дефолт боюнча OpenServerде пароль бош болот
$dbname = "test_db";

// Туташууну түзүү
$conn = new mysqli($servername, $username, $password, $dbname);

// Текшерүү
if ($conn->connect_error) {
    die("Базага туташуу ишке ашкан жок: " . $conn->connect_error);
}
echo "Маалымат базасына туташуу ийгиликтүү аткарылды!";
?>

10.3. SQL Инъекциядан коргонуу жана Коопсуз маалымат кошуу (Prepared Statements)

Колдонуучулар киргизген зыяндуу коддордон коргонуу үчүн түз SQL суроонун ордуна даярдалган суроолорду (Prepared Statements) колдонуу зарыл:

<?php
// Формадан келген маалыматтар
$name = $_POST['name'] ?? 'Белгисиз';
$age = $_POST['age'] ?? 0;

// Суроону даярдоо
$stmt = $conn->prepare("INSERT INTO users (name, age) VALUES (?, ?)");
// Параметрлерди байлоо ("si" - биринчиси string, экинчиси integer)
$stmt->bind_param("si", $name, $age);

// Аткаруу
if($stmt->execute()) {
    echo "Жаңы колдонуучу базага коопсуз кошулду!";
} else {
    echo "Ката келип чыкты: " . $stmt->error;
}

$stmt->close();
$conn->close(); // Байланышты үзүү
?>

10.4. Маалыматтарды жаңыртуу (UPDATE)

Базадагы бар болгон маалыматты өзгөртүү үчүн төмөнкү логика колдонулат:

<?php
$new_age = 26;
$target_name = "Uluk";

$stmt = $conn->prepare("UPDATE users SET age=? WHERE name=?");
$stmt->bind_param("is", $new_age, $target_name);
$stmt->execute();

echo "Колдонуучунун маалыматы ийгиликтүү жаңыртылды!";
$stmt->close();
?>
